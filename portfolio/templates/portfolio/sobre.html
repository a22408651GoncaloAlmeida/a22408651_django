Sobre esta Aplicação

1. Arquitetura MVT do Django

O Django segue o padrão MVT (Model-View-Template), uma variante do padrão MVC adaptada ao framework.

Model

Define a estrutura dos dados e comunica com a base de dados. Cada modelo corresponde a uma tabela na BD. Exemplos: Projeto, Tecnologia, UnidadeCurricular.

View

Contém a lógica de negócio. Recebe um pedido HTTP, interage com os modelos, e devolve uma resposta — normalmente renderizando um template com dados.

Template

Ficheiros HTML com linguagem de template Django. Recebem dados da view e geram o HTML final enviado ao browser.

Fluxo de informação

  1. Browser envia um pedido HTTP (ex: GET /portfolio/projetos/)
  2. urls.py faz o mapeamento para a view correspondente
  3. View consulta o Model para obter os dados necessários
  4. Model executa queries à base de dados e devolve os dados
  5. View passa os dados ao Template
  6. Template gera o HTML final
  7. Browser recebe e renderiza a resposta HTML

class Projeto(models.Model):
    nome = models.CharField(max_length=100)
    descricao = models.TextField()
    ano = models.IntegerField()
    unidade_curricular = models.ForeignKey('UnidadeCurricular', ...)
    tecnologias = models.ManyToManyField('Tecnologia', ...)

Como foi utilizado

O GitHub foi usado para versionar o código ao longo do desenvolvimento, permitindo guardar o histórico de alterações e recuperar versões anteriores em caso de erro. Cada funcionalidade foi desenvolvida e depois commitada com uma mensagem descritiva.

Vantagens

  • Histórico completo de todas as alterações feitas ao código
  • Possibilidade de reverter para versões anteriores
  • Integração com GitHub Codespaces para desenvolvimento na cloud
  • Facilita colaboração em equipa
  • Backup automático do código
